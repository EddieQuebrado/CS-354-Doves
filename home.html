History of Python

Python was made by Guido can Rossum in 1989 at CWI! No, not that CWI. Cebtrum Wiskunde and Informatica, in Amserdam. Guido built the language, inspired by languages like Java, Lisp, Perl, Haskell and C, to try to create something just as powerful and useable, while also making it as easy-to-read and easy-to-write as possible. Bringing powerful languages down to a level where a less skilled programmer can still weild all the power a language like C++ or Java can have, without all the annoying intricacies or weird formatting to hinder them.

The language is known for being a bit of a one-size-fits-all language, useable in a lot of different programming situations. Scripting, programming, machine learning, graphics processing, and all kinds of other things. However, instead of making the language with all this functionality out-of-the-box, Guido decided to instead focus on making Python an incredibly extensible language. Combined with it's focus on ease of use, as well as readaiblity, it makes Python one of the most popular and most versitile language in the world.

According to 'Zen of Python', written by Tim Peters, Python has many key guiding principles:

  • Beautiful is better than ugly.
  • Explicit is better than implicit.
  • Simple is better than complex.
  • Complex is better than complicated.
  • Flat is better than nested.
  • Sparse is better than dense.
  • Readability counts.
  • Special cases aren't special enough to break the rules.
  • Although practicality beats purity.
  • Errors should never pass silently.
  • Unless explicitly silenced.
  • In the face of ambiguity, refuse the temptation to guess.
  • There should be one—and preferably only one—obvious way to do it.
  • Although that way may not be obvious at first unless you're Dutch.
  • Now is better than never.
  • Although never is often better than right now.
  • If the implementation is hard to explain, it's a bad idea.
  • If the implementation is easy to explain, it may be a good idea.
  • Namespaces are one honking great idea—let's do more of those!
  • Use proper indentation or I'll break your kneecaps.

(...Ok, maybe not that last one.)

So, while it's not the most technically powerful language in the world, it's extensibility combined with it's user-friendly nature makes it one of the most effective languages in the world.

Where Python is used today

Have you ever used Wikipedia? How about Google? Facebook? Spotify? Reddit? Well, congradulations, all of those sites use python in their websites. In fact, Reddit is implemented in Python entirely! It's versatility makes it a natural choice for most websites.

Python is included in standard operating system MacOS, as well as almost every distribution of Linux.

Python is even used by NASA! Meeting the United Space Alliance's requirements of a language being 'fast, cheap and light', Python is used by NASA today to do secret-NASA things. They don't tell anyone exactly what they're doing with it, but they're NASA, so that makes sense.

The Raspberry Pi (a computer about the size of a wallet, costing less than $40) has the word 'Pi' in it's name not only as a clever play on words, but as a reference to Python! Python is a primary language used on the Pi, and a majority of the systems on it are built and optimized for Python programs and programming.

Python has even gone on to inspire many other languages, such as Groovy, a language built to make Java more Python-like. Other inspired languages include Cobra, CoffeeScript, Go, Swift, and many others. Specifically, those languages usually follow in Python's footsteps in terms of syntax, indentation style, and ease-of-use.

Python's programming paradigms

Python supports four main programming paradigms: imperative, functional, procedural, and object oriented. This page will only be highlighting the object oriented and procedural paradigms.

Object oriented Python

Python is used as an object-oriented language (OOL). Object-oriented programming is a unique and difficult concept, but to put it simply: it acts on the idea that everything in a given program is an 'object', with properties and functions that can be the same as other objects, but act independantly. It saves on resource useage, while also making the program much easier to manage. In an OOL, if you want there to be two different cars in your program, you just call two Cars(), and differentiate them. In a non-OOL, however, you have to create the codebase and the programming for each car independantly, which takes more time, disk space, and resource useage when running the program. This is a simple explanation, of course.

Procedural Python

Python is commonaly used as a procedural programming language in scripting environments. This paradigm is a subset fo imperative programming in which statements are structured into procedures. This paradigm is typically a list of instructions that execute one after the other starting from the top of the first line of the program. This is useful for creating modules that can be unrelated to other modules and programs yet still be accessed by said programs and work together with other modules to achieve goals for the program. As well as support resue of procedures within a program that are defined in either other libraries or modules.

Quirks and and advantages

No language is perfect, but every language has it's specialties, and Python is no exception. What some would consider downsides, other consider features. Here's a few examples of how Python is a bit different than other, similar languages:

  • It's extreme Popularity makes it a bit of a token choice at times, but also makes it compatible with countless codebases and modules people have made in the past
  • It's fundamentally Object-Oriented nature makes it useable in modern object-oriented programming styles, and especially powerful when dealing with large amounts of objects or data.
  • Python features Automatic Garbage-Collection, making it easy for people to toss large amounts of data around without having to worry about data leaks.
  • It's Ease-of-Use as well as it's relatively Forgiving Learning Curve makes it ideal for beginning programmers, or those learning the language for a project.
  • It's Expansive Standard Library makes it incredibly powerful even without any expansions, making it incredibly useful out-of-the-box.
  • Python is an Indentation/Whitespace Sensitive language, meaning the indentation style is important and fundamental to how it's written. Many people don't like this, but it removes the need for many formatting items such as brackets ((){}[]) in certain scenarios.
